# 프로젝트 케이스 발굴

> 트리거: "케이스 작성", "프로젝트 정리", "성과 발굴", "HIH_2에서 찾아줘"

## 절차

### Step 1: HIH_2 원본 탐색

대상 디렉토리: `~/projects/HIH_2/`

주요 참조 소스:
- `HIH_Claude/보고서/` — 기술 분석 보고서 (00~18번)
- `HIH_Claude/산출물/` — ICD, SRS, SyRS, 코드리뷰
- `HIH_Claude/데이터/` — CSV 데이터, Python 스크립트
- `HIH_Claude/참고자료/` — 부품, WBS, 회의록
- `HIH_Claude/task_gint.md` — 작업 이력 (정량적 성과 추출)
- `1)프로젝트/5)농업/` — 프로젝트 원본 파일

### Step 2: 정량적 성과 추출

찾아야 할 것:
- **수치**: 몇 건, 몇 %, 몇 시간 절감, 몇 개 관리
- **Before/After**: 기존 방식 vs 개선 후
- **의사결정**: 왜 이 방식을 선택했는가 (대안과 비교)
- **문제 해결**: 어떤 문제 → 어떤 분석 → 어떤 해결

### Step 3: 스토리 구성

포맷:
```
## [프로젝트/기술 이름]

### 상황 (Situation)
- 어떤 제품/프로젝트인지, 본인의 역할

### 과제 (Task)
- 해결해야 할 핵심 문제, 제약 조건

### 행동 (Action)
- 제1원칙 관점에서 어떻게 접근했는가
- 기술적으로 무엇을 했는가 (구체적)

### 결과 (Result)
- 정량적 성과 (수치 필수)
- 비즈니스 임팩트
```

### Step 4: 사용자 검토

- 작성된 케이스를 사용자에게 브리핑
- 사실 관계 확인 (수치, 역할, 기여도)
- 톤/강조점 조율 후 확정
