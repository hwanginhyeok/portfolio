## Part 1. 기술 전문가 리뷰

### 강점
1. **시스템 레벨 사고**: 단일 모터 제어가 아니라 VCU↔배터리↔모터↔펌프↔자율주행의 전체 시스템을 이해하고 설계
2. **CAN 통신 설계 실무**: DBC 파일을 직접 작성하고, 다중 노드(VCU, 모터컨트롤러, BMS, 팬) 간 정합성을 검증한 경험
3. **배터리 BMS 정합성**: SOC 오류(162.4%) 같은 실제 필드 이슈를 데이터 기반으로 분석하고 3차까지 반복 검증
4. **상태머신 설계**: RC/LCD/ADT 3가지 제어 모드 간 전환 로직을 설계하고, 비정상 전환 시나리오까지 커버
5. **직무발명 특허**: 모터 초기 위치 검출 알고리즘 — 기술 깊이를 증명하는 강력한 근거

### 추가 발굴 강점 (에이전트 탐색 결과)
6. **범퍼 안전 시스템 정량 검증**: 3km/h에서 정지거리 0.082m, 상전류 피크 308A, 정착시간 467ms — 수치 기반 안전성 입증
7. **근본원인 분석(RCA) 역량**: 동파 문제에서 솔레노이드 오링 가공불량(depth+wall thickness)까지 추적, 5호기 동시 발생 패턴 분석
8. **비상정지 로직 재설계**: Main Relay 소손 원인을 분석하고, "VCU 정지명령 → 5분 대기 → MR 차단" 안전 시퀀스를 설계
9. **동적 시험 수행**: BLACKTAN 테스트(Inrush/정격/과부하/회전/무부하 5종) 직접 수행
10. **부품 132개 항목 관리**: 양산 BOM V0.3.4까지 진화, 재고 미확보 23건 식별

### 약점 / 보완 필요
1. **FOC 구현 상세 누락**: BLDC 제어라고만 되어있고, FOC/사다리꼴 구동 중 어떤 방식인지, Clarke/Park 변환, PI 게인 튜닝 경험이 드러나지 않음
2. **정량적 성과 부족**: "이슈 37건 관리"는 있지만, "해결율 몇 %", "개선 전후 비교" 같은 Before/After 스토리가 약함
3. **코드 수준 역량 미노출**: 상태머신, CAN 프로토콜 파싱, 제어 로직을 직접 짠 건지 리뷰만 한 건지 불명확

### 예상 질문 & 답변

**Q1. FOC 알고리즘을 직접 구현해본 경험이 있나요? Clarke/Park 변환, 전류 루프 튜닝 과정을 설명해주세요.**

> **A1**: SS500에서 주 구동모터는 외부 모터컨트롤러(DMKE 등)를 CAN으로 제어하는 구조라 FOC를 직접 MCU에 구현하진 않았습니다. 하지만 SCU(Steering Control Unit) 프로젝트에서 Infineon TC23x 기반으로 BLDC 사다리꼴 구동을 구현했고, 직무발명 특허인 '모터 초기 위치 검출'은 FOC의 전제 조건인 로터 위치 추정에 대한 것입니다. Clarke/Park 변환의 수학적 원리와 d-q축 전류 제어 구조는 이해하고 있으며, 이를 기반으로 모터컨트롤러 벤더와 CAN 파라미터(전류 제한, 가감속 시간, 속도 피드백 주기)를 협의했습니다.
>
> **디벨롭 포인트**: SCU 프로젝트의 BLDC 구동 경험 + 특허를 연결해서 "FOC의 원리를 이해하고, 시스템 레벨에서 모터 제어를 설계한 경험" 으로 프레이밍

**Q2. CAN 통신 설계에서 가장 어려웠던 문제와 해결 방법은?**

> **A2**: 가장 어려웠던 건 배터리 BMS와의 CAN 정합성 문제입니다. 배터리 업체가 ROM 업데이트를 해도 SOC가 162.4%로 표시되고 보호 로직(Warning/Fault/Protection Flag)이 작동하지 않는 상황이 반복됐습니다.
>
> 접근 방법: (1) CAN 로그(.asc)를 분석해서 셀별 전압 편차(7번 셀 3.34V vs 평균 3.46V, 최대 3.60V)를 정량화하고, (2) CAN DB 정합성 검토 시트를 만들어 신호별 기대값 vs 실측값을 비교하고, (3) Kill Switch 시퀀스와 Power Hold 로직까지 포함한 통합 검증 체크리스트를 만들었습니다. 3차에 걸친 검증(25.12, 26.01, 26.02)을 통해 배터리 업체에 구체적 요구사항을 전달하고 WBS까지 수립해서 관리했습니다.
>
> **디벨롭 포인트**: "데이터 기반 문제 분석 → 체계적 검증 프로세스 수립 → 외부 업체 관리"의 STAR 스토리로 강화

**Q3. 48V 시스템에서 전력 예산(Power Budget)은 어떻게 관리했나요?**

> **A3**: SS500의 전체 전력 예산은 펌프 성능 이슈(ISSUE #60)를 분석할 때 체계적으로 정리했습니다.
> - 구동모터: 사양별 상이 (CAN 전류 제한으로 관리)
> - 펌프: 현재 0.96kW → 타겟 3.23kW (3.4배 증가)
> - 팬모터: Hobbywing M6215 기반
> - 보조기기: DC-DC 12V 변환, LCD, 센서류
>
> 배터리 460Ah 용량 대비 연속 운전 시간을 현 4시간에서 유지하면서도 펌프 성능을 올려야 하는 트레이드오프가 핵심 과제였고, 80A급 펌프 적용 시 2시간으로 줄어드는 문제를 정량적으로 분석해서 경영진에 보고했습니다.
>
> **디벨롭 포인트**: Power Budget 분석표를 시각화 자료로 제작 → 포트폴리오에 삽입

**Q4. 상태머신 설계 원칙은? 비정상 상태 전환 시 어떻게 안전을 확보했나요?**

> **A4**: SS500 VCU의 상태머신은 3가지 제어 모드(RC/LCD/ADT)와 기능안전 상태(비상정지, Main Relay 차단)를 통합 관리합니다.
>
> 제어권 전환 설계 원칙:
> 1. **모든 Transition에서 Init 루틴 실행** — 제어권 전환 시 모든 모터 컨트롤러에 속도 0 지령 인가 (ISSUE #79에서 실차 검증)
> 2. **NONE 상태 제거** — 기존에 "제어권 없음" 상태가 있어서 모터가 이전 속도를 유지하는 버그 발견 → NONE 삭제하고 항상 명시적 제어 모드 유지
> 3. **비상정지 최우선** — Kill Switch → BMS Shutdown 요청 → Main Relay 차단 시퀀스 설계, 5분 알람 + 5분 대기 후 강제 OFF
>
> **디벨롭 포인트**: 상태 전이도(State Transition Diagram)를 직접 그려서 포트폴리오에 포함

## Part 2. 인사담당자(HR) 리뷰

### 강점
1. **크로스펑셔널 역량 명확**: 전력제어(기술) + PM(관리) 이중 역할을 수치로 증명 가능 (이슈 14건 단독/공동 담당)
2. **체계적 문서화 습관**: Notion DB 기반 이슈 트래킹, CAN 정합성 검토 시트, WBS 등 프로세스를 만들어 운영
3. **글로벌 경험**: 일본 전시회 샘플 제작(ISSUE #77), 중국 부품 업체 서베이(PLANT MATE 등)
4. **AI 도구 활용**: Claude Code 기반 개발 자동화, Notion 연동 워크플로우
5. **특허 보유**: 직무발명 특허 출원 — 기술 창의성 증명

### 약점 / 보완 필요
1. **커리어 내러티브 부재**: "왜 농기계?"에서 "왜 Physical AI?"로의 전환 스토리가 없음
2. **리더십 경험 부족**: PM이라고는 하지만 "주니어 PM" — 몇 명을 리드했는지, 의사결정 권한 범위가 불명확
3. **협업 에피소드 미노출**: 융합제어팀, HW팀, MD팀과의 협업 과정에서 갈등/조율 사례가 없음

### 예상 질문 & 답변

**Q1. 왜 농기계 회사에서 일했나요? 앞으로의 커리어 방향은?**

> **A1**: GINT에서의 경험은 "AI가 물리 세계와 만나는 접점"을 가장 밀도 높게 체험할 수 있는 기회였습니다. SS500은 단순한 농기계가 아니라 — 48V 전동 파워트레인, CAN 기반 분산 제어, 자율주행(GNSS+카메라+IMU), 클라우드 연동까지 갖춘 Physical AI 제품입니다.
>
> 실제로 GINT는 Foresight(이스라엘 3D 비전), KONEC(한국 Tier-1)과 자율주행 트랙터 공동개발 중이고, 2026년부터 양산 판매를 시작합니다. 이 과정에서 모터 제어부터 시스템 아키텍처, APQP 양산 프로세스까지 제품 개발의 전체 생명주기를 경험했습니다.
>
> 앞으로는 이 경험을 로보틱스, 자율주행, 스마트 디바이스 등 Physical AI가 확장되는 영역에서 활용하고 싶습니다. "하드웨어를 이해하면서 소프트웨어도 쓸 수 있는 브릿지 엔지니어"가 제 포지셔닝입니다.
>
> **디벨롭 포인트**: "농기계" → "Physical AI 제품"으로 프레이밍. GINT의 글로벌 협업 맥락을 반드시 언급

**Q2. PM으로서 가장 어려웠던 상황과 어떻게 해결했나요?**

## Part 3. CEO 리뷰

### 강점
1. **제품 사고**: 기술 하나가 아니라 "제품이 시장에 나가기까지"의 전체 과정을 경험

3. **공급망 관리**: 국내/중국 업체 서베이, 프로포마 인보이스 관리, 리드타임 확인 등 실무 경험
4. **스타트업 DNA**: 소규모 조직에서 여러 역할을 동시에 수행한 적응력

### 약점 / 보완 필요
1. **비즈니스 임팩트 연결 부족**: 기술 성과는 있지만 "그래서 회사에 얼마나 기여했나?"가 약함
2. **스케일 경험 부재**: 양산 "준비"까지는 했는데, 실제 양산 라인 운영/품질 관리 경험은?
3. **전략적 사고 증명 부족**: CEO는 "이 사람이 팀장/리드가 되면 어떤 의사결정을 할까"를 봄

### 예상 질문 & 답변

**Q1. SS500 프로젝트에서 당신이 없었으면 어떤 일이 벌어졌을까요?**

> **A1**: 세 가지가 달라졌을 겁니다:
> 1. **배터리 정합성 이슈가 늦게 발견됐을 것**: 배터리 업체의 ROM 업데이트 후 SOC가 162%로 표시되는데도 보호로직이 안 도는 문제를 제가 CAN 로그를 분석해서 발견하고, 셀별 전압 편차까지 정량화해서 업체에 전달했습니다. 이걸 놓치고 양산에 들어갔으면 필드에서 배터리 안전 사고가 날 수 있었습니다.
> 2. **제어권 전환 시 안전 버그**: RC↔LCD 전환 시 모터 속도가 초기화되지 않는 버그를 발견하고, NONE 상태를 삭제하는 아키텍처 변경을 제안/실행했습니다. 이건 사용자 안전 직결 이슈였습니다.
> 3. **펌프 성능 업그레이드 의사결정 지연**: 기존 20Bar/23LPM에서 25Bar/40LPM으로 올리려면 비용 3배, 운전시간 반감이라는 트레이드오프가 있었는데, 이걸 국내외 업체 서베이 + 정량 분석으로 정리해서 경영진 의사결정을 도왔습니다.
>
> **디벨롭 포인트**: "없었으면" 질문은 본인의 불가대체성을 보여주는 최고의 프레임. 안전 이슈 방지 → 비즈니스 리스크 감소로 연결

**Q2. Physical AI 시장에서 우리 회사가 어떤 인재가 필요하다고 생각하나요?**

> **A2**: 2026년 Physical AI 시장은 CAGR 38.5%로 성장하고 있고, CES 2026에서도 핵심 키워드였습니다. 하지만 대부분의 회사가 겪는 문제는 "AI 알고리즘은 있는데 물리적 제품으로 못 만든다"는 것입니다.
>
> 필요한 인재는:
> 1. **시스템 통합 엔지니어**: AI → 임베디드 → 액추에이터 → 센서의 전체 루프를 이해하는 사람
> 2. **양산 경험**: 프로토타입에서 양산까지 APQP 프로세스를 실행해본 사람
> 3. **규제/인증 대응**: 농기계 종합검정, 안전 기능(비상정지, 펌퍼 시스템) 등 실제 인증 경험
>
> 저는 이 세 가지를 SS500 한 프로젝트에서 모두 경험했습니다.
>
> **디벨롭 포인트**: "시장 이해 + 자기 포지셔닝"을 한 번에 보여주는 답변 구조

**Q3. 당신이 팀 리드라면 SS500을 어떻게 더 잘 만들었을까요?**

> **A3**: 일론 머스크의 개발 5단계로 답하겠습니다:
> 1. **요구사항을 의심**: 현재 아세아텍 100A급 벤치마킹 기준이 맞는지? 우리 타겟 고객(과수원 규모, 작업 패턴)에 맞는 스펙인지 재검토
> 2. **삭제**: 현재 3가지 제어모드(RC/LCD/ADT) 중 LCD 모드가 정말 필요한지? 제어모드를 줄이면 상태머신 복잡도가 극적으로 감소
> 3. **단순화**: CAN 메시지를 표준화하고, 모터 컨트롤러 벤더를 단일화해서 DBC 관리 복잡도 감소
> 4. **속도**: 배터리 업체와의 정합성 검증을 3차가 아니라 1차에 끝내기 위해, 자동화된 CAN 테스트 벤치 구축
> 5. **자동화**: 이미 Claude Code로 이슈 관리/주간보고를 자동화했지만, 시험 데이터 수집→분석→보고서까지 E2E 자동화
>
> **디벨롭 포인트**: "5단계 프로세스"를 실제 프로젝트에 적용한 구체적 사례 — 포트폴리오의 핵심 스토리라인으로 활용
