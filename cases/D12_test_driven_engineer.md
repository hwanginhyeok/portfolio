# Case Study: 확인하고 싶으면 장비를 만들어서라도 본다

> "측정할 수 없으면 개선할 수 없다" — 피터 드러커
> "시험 방법이 없으면 만든다. 데이터가 없으면 만든다. 감으로 하지 않는다."

---

## 마인드셋

엔지니어링에서 가장 위험한 말은 "대충 될 것 같다"입니다.

저는 확인하고 싶은 것이 있으면, 시험 방법이 없어도 **장비를 만들어서라도 봅니다**. 이것은 취미가 아니라 원칙입니다. 3년간 9개의 시험 체계를 직접 설계/구축/운영하면서, 매번 같은 루프를 반복했습니다:

```
가설 수립 → 측정 방법 설계 → 장비/벤치 구축 → 데이터 수집 → 정량적 결론 → 의사결정
```

아래는 시간순으로 엮은 그 여정입니다.

---

## Phase 1. 기초를 쌓다 (2023)

### 1. 다이나모미터 토크제어 시스템 — "토크를 전압으로 변환하다"

**가설**: 모터 성능 평가를 위해 정밀한 토크 측정이 필요한데, 기존 다이나모미터의 토크 제어가 불안정하다.

**만든 것**:
- Strain Gauge 기반 Full Bridge Circuit 설계
- Wheatstone Bridge 아날로그 신호 처리 회로
- 토크 컨트롤러 펌웨어 개발
- 10포인트 교정 체계 (0~9.811 dN·m)

**정량적 결과**:
| 항목 | 수치 |
|------|------|
| 토크 감도 | 0.996 N·m |
| 비선형성 | **0.008%** |
| 교정 포인트 | 10개 (완벽 교정) |
| 추가 성과 | **사내 기술 세미나 개최** (질의응답 포함) |

**임팩트**: 이후 EOP 400W, ADT 모터의 모든 성능 시험에서 이 다이나모미터가 표준 시험 장비로 사용됨. "만든 장비가 회사의 인프라가 된" 첫 사례.

---

## Phase 2. 방법론을 확장하다 (2024)

### 2. PWM 절환 온도 비교 시험 — "16°C의 차이를 숫자로 증명하다"

**가설**: SVPWM이 기존 DPWM 대비 FET 온도가 낮을 것이다. 하지만 "낮다"만으로는 부족하다. **얼마나** 낮은지 정량화해야 한다.

**만든 것**:
- SVPWM vs DPWM 비교 시험 프로토콜 설계
- 온도 조건별(55°/85°/120°) 시험 매트릭스
- 실시간 데이터 로깅 체계 (CSV 출력)

**정량적 결과**:
| 항목 | 수치 |
|------|------|
| 데이터 포인트 | **2,932개** (48분 52초 연속 수집) |
| FET 온도 저감 | **16~17°C** (SVPWM 적용 시) |
| 시험 조건 | 55°/85°/120° × 2개 Assy × 2개 PWM 방식 = **12세트** |

**임팩트**: 국책과제(EOP 400W) 완료보고서의 핵심 근거 데이터. "SVPWM 적용이 열 성능을 개선한다"는 주장을 12세트 비교 시험으로 입증.

### 3. ADT 모터 실주행 비교 — "벤치가 아닌 실제 도로에서 증명하다"

**가설**: 3세대 모터 개선(TVS Diode 적용, PI 파라미터 최적화)이 실제 주행에서도 효과가 있을 것이다.

**만든 것**:
- 실주행 데이터 수집 → 개선 전/후 비교 분석 프레임워크
- Zbench 지표 기반 정량 평가 체계

**정량적 결과**:
| 항목 | Before | After | 개선 |
|------|--------|-------|------|
| Zbench 지표 | 2.714 | 4.346 | **+60%** |
| 오버슈팅(OVT) | 발생 | **제거** (TVS Diode) | 100% |
| 리플 안정성 | 39~147 ppm (변동 큼) | 139~142 ppm | **안정화** |

## Phase 3. 시스템으로 진화하다 (2025~2026, SS500)

SS500 프로젝트에서는 단일 시험이 아니라, **제품 전체를 커버하는 시험 체계**를 구축했습니다.

### 4. 펌프 성능평가 시스템 — "상용 시험기가 없으면 직접 만든다"

**가설**: 펌프 3개 시료 중 어떤 것이 최적인지, 그리고 압력-유량 관계가 선형인지 확인해야 한다. 하지만 48V 전동 펌프용 성능평가 시스템은 시장에 없다.

**만든 것**:
- Blancett B2800 유량계 + SENSYS PSH 압력계 조합
- 배관 + 밸브 + 탱크 + 계측 구성
- 테스트베드 **단독 구축** (ISSUE #126)
- 시험 일정: 구축(~02/14) → 시운전(~02/21)

**정량적 결과**:
| 시료 | 유량 (L/min) | 비고 |
|------|-------------|------|
| 시료 A | 10.4 | 최저 |
| 시료 B | 13.0 | 중간 |
| 시료 C | 15.6 | 최고 |
| **발견** | **압력-유량 선형관계 규명** | 펌프 선정 기준 수립 |

**임팩트**: 3개 시료의 성능 차이를 정량적으로 비교하여 펌프 선정 의사결정의 근거를 마련. "감성 평가"가 아닌 "데이터 기반 선정".

### 5. 팬 시스템 벤치 — "80% 갭 축소의 효과를 숫자로 보여주다"

**가설**: 프로펠러 크기를 키우고(18→20파이) 슈라우드 갭을 줄이면(45→9mm) 풍량이 증가할 것이다. 하지만 "체감상 세다"로는 부족하다.

**만든 것**:
- 카메라 스탠드 + 클램프 + 프로펠러형 풍속계(~50m/s) 조합
- 거리별(1, 2, 3, 4, 5m) 풍속 측정 체계
- RS-500X(기계식 기성 모델) 대조군 포함
- 감성평가 + 정량평가 **동시 수행** 프로토콜

**정량적 결과**:
| 항목 | Before | After | 변화 |
|------|--------|-------|------|
| 프로펠러 직경 | 18파이 | 20파이 | +11% |
| 슈라우드 갭 | 45 mm | 9 mm | **-80%** |
| 모터 출력 | 19.7 W | 42.2 W | **+114%** |
| 모터 전류 | 16.74 A | 38.7 A | +131% |
| 감성평가 | - | "풍압 증가 감지됨" | 정성 확인 |

**임팩트**: 갭 축소 설계 변경의 효과를 Before/After로 입증. 기구팀에 모터 플레이트 변경(4t→12t) + 어댑터(5t) 추가 설계를 정량적 근거와 함께 요청.

### 6. 범퍼 안전 시험 — "0.082m를 숫자로 증명하다"

**가설**: 범퍼 감지 시 차량이 충분히 빠르게 정지하는가? 자율주행 안전의 핵심.

**만든 것**:
- 속도별 정지거리 **6단계 이론 모델** 수립 (0.5~3.0 km/h)
  - 물리 공식: 정지거리 = v² / (2a), a = 0.5167 m/s²
- 범퍼 스트로크(123.5mm), 센서 인식거리(17.5mm) 기반 설계 사상 정의
- 실차 시험 → 이론값 vs 실측값 비교

**정량적 결과**:
| 속도 (km/h) | 이론 정지거리 (m) | 실측 정지거리 (m) | 정지시간 | 피크 전류 |
|------------|-----------------|-----------------|---------|----------|
| 3.0 | 0.672 | **0.082** | ≤0.05초 | **308 A** |
| 2.0 | 0.299 | **0.055** | - | 158.8 A |
| 1.0 | 0.075 | **0.028** | - | -156.5 A |

**핵심 발견**: 이론 모델(등가속 가정)은 실측 대비 **8배 이상** 보수적. 실제로는 전기 제동(308A 피크)이 기계적 관성보다 훨씬 빠르게 작용. 브레이크 전류 정착시간 467ms도 정량화.

**임팩트**: 자율주행 안전 인증의 핵심 데이터. "범퍼가 있다"가 아니라 "3km/h에서 0.082m 내에 정지한다"고 말할 수 있게 됨.

### 7. SS500 vs 기성모델 성능 비교 — "경쟁 제품을 빌려서라도 비교한다"

**가설**: SS500이 기계식 기성모델(한아 SS기) 대비 어떤 점에서 우위/열위인지 정량적으로 파악해야 한다.

**만든 것**:
- 기성모델 실물 대여 (대여기간 ~01/08)
- 노즐 시스템, 펌프 시스템, 팬 시스템 **3대 축** 비교 프레임워크
- 분사 성능 정성적 비교: 노즐 18ea vs 36ea 실제 분사 영상 촬영

**정량적 결과**:
| 항목 | SS500 | 기성모델(한아) | 비교 |
|------|-------|-------------|------|
| 펌프 압력 | **30 bar** | 살포 15 / 급수 25 bar | **SS500 2배** |
| 펌프 유량 | 24 L/min | 99 L/min | 기성모델 4배 |
| 노즐 수 | 12 | 36 | 기성모델 3배 |
| 송풍 회전수 | ~8000 rpm | ~3000 rpm | SS500 2.7배 |
| 사용자 제어성 | 3단계 + 개별 제어 | 로터리 + 연동 | **SS500 우위** |

**핵심 발견**:
- 노즐 18ea(17bar) vs 36ea(12bar) → **노즐 수와 압력은 역관계**
- SS500은 압력 우위(2배), 기성모델은 유량 우위(4배)
- SS500의 차별점은 **전자식 개별 제어** (농약 안 맞으면서 조절 가능)

### 8. 배터리 CAN 정합성 검증 체크리스트 — "신뢰하되, 검증하라"

**가설**: 배터리 업체가 "수정 완료"라고 한 ROM이 실제로 정합성을 만족하는가?

**만든 것**:
- 신호별 기대값 vs 실측값 자동 비교 시트
- 검증 체크리스트 (CAN DB / Kill Switch / Power Hold / SOC / Warning-Fault-Protection)
- 3차 반복 검증 프로토콜 (25.12 → 26.01 → 26.02)

**결과**: D-2 배터리 케이스 스터디 참조 (SOC 162% 발견 → 업체 WBS 수립)

### 9. 내구성 시험 설계 — "128,000 사이클을 견딜 수 있는가"

**가설**: 양산 전에 SS500의 핵심 부품(펌프, 솔레노이드, 교반 시스템)이 실사용 조건에서 충분한 내구성을 확보해야 한다.

**만든 것**:
- **128,000 Cycle Kill Test** 시험 계획 수립
- Worst Case 교반 Profile 정의
- 극한 조건 매트릭스: 온도(고온/저온) × 습도(고습/건조) × 지형(평지/경사)
- 내구 시험 검토서 작성 (20251209)

**임팩트**: 양산 품질 확보의 기초. "프로토타입이 동작한다"와 "양산품이 3년 쓴다"는 전혀 다른 문제이며, 그 차이를 메우는 것이 내구성 시험.

## 패턴 — 반복되는 루프

9개 사례를 관통하는 패턴:

```
Phase 1 (2023): 단일 장비 제작 — 다이나모미터
    ↓ "장비를 만들면 데이터가 나온다"
Phase 2 (2024): 비교 시험 방법론 — PWM 온도, ADT 실주행
    ↓ "Before/After를 숫자로 보여주면 설득된다"
Phase 3 (2025~): 시스템 시험 체계 — 펌프/팬/범퍼/내구/정합성/성능비교
    ↓ "제품 전체를 커버하는 시험 인프라"
```

매번 같은 루프: **가설 → 측정 방법 설계 → 장비 구축 → 데이터 수집 → 정량적 결론**

이것은 "시험을 좋아해서"가 아닙니다. **"데이터 없이 의사결정하는 것이 가장 비싼 비용"**이라는 것을 경험으로 알기 때문입니다.

## 면접 예상 Q&A

**Q1. 시험장비를 직접 만드는 건 비효율적이지 않나요? 외주를 주면 되잖아요.**

> 48V 전동 농기계용 펌프 성능평가 시스템은 시장에 존재하지 않습니다. 외주를 줘도 우리 제품의 운전 조건(48V, 10~40LPM, CAN 연동)을 이해하는 업체를 찾기 어렵고, 찾더라도 리드타임이 수개월입니다. 제가 2주 만에 구축한 펌프 벤치가 외주 대비 빠르고, 정확하고, 반복 사용 가능했습니다.
>
> 더 중요한 건, **시험 장비를 직접 만들면 "무엇을 측정해야 하는지"에 대한 이해가 깊어진다**는 것입니다. 다이나모미터를 만들면서 토크 측정의 원리를 체득했고, 그 이해가 이후 모든 모터 평가에 적용됐습니다.

**Q2. 가장 만족스러운 시험 결과는?**

> 범퍼 안전 시험입니다. 이론 모델에서는 3km/h 정지거리가 0.672m로 계산됐는데, 실측은 0.082m — 8배 이상 좋았습니다. 이론(등가속 가정)과 실제(전기 제동)의 차이를 정량적으로 확인한 순간이었고, 동시에 "전동 구동의 강점이 여기에 있다"는 확신을 얻었습니다. 308A 피크 전류로 0.05초 만에 정지하는 전기 제동은 기계식 브레이크와 차원이 다릅니다.

**Q3. 데이터가 기대와 다를 때 어떻게 하나요?**

> SOC 162% 사례가 정확히 그런 경우입니다. "보호 로직이 동작해야 한다"는 기대와 "아무런 Warning도 발생하지 않는다"는 실측이 완전히 반대였습니다. 이때 중요한 건 **데이터를 의심하지 않는 것**입니다. "장비가 잘못된 건 아닌가?" 대신 "왜 이 값이 나왔는가?"를 먼저 물어봅니다.
>
> SOC 162%의 경우, 셀별 전압(7번 셀 3.34V vs 평균 3.46V)까지 파고 들어가서 BMS의 SOC 계산 로직 자체에 문제가 있음을 확인했습니다. 데이터를 믿고, 데이터의 원인을 추적하는 것이 원칙입니다.
