# About Me — 커리어 내러티브

> 소프트웨어가 물리 세계와 만나는 접점에서 제품을 만드는 사람

---

## 한 줄 소개

모터제어 개발자이자 주니어 PM. 하드웨어를 설계하면서 소프트웨어를 이해하고, 기술을 깊이 파면서도 제품 전체를 조망하는 **브릿지 엔지니어**입니다.

---

## 지금까지의 여정

### 모터제어 개발자로 시작하다

GINT(긴트)의 전력제어개발팀에서 BLDC 모터 제어 개발자로 커리어를 시작했습니다. 첫 프로젝트는 자동차 Tier-1 부품인 SCU(Steering Control Unit) 개발이었습니다. Infineon TC23x MCU 위에서 BLDC 사다리꼴 구동을 구현하고, 자동차 환경시험(ES95400)/EMC(ES96200) 규격을 통과시키는 과정에서 "양산 품질이 무엇인지"를 체감했습니다. DIC(현대) 협력사로서 CAN 통신 설계, PCB 3세대 진화(ES→AS→A2S), SBW 부하시험까지 — 단순히 코드를 짜는 것이 아니라, 제품이 현실 세계에서 동작하도록 만드는 일이었습니다.

이 시기에 **직무발명 특허 2건**을 출원했습니다:
- **모터 초기 위치 검출 알고리즘** (PN231067KR, 2023) — 센서리스 BLDC 구동의 전제 조건인 로터 초기 위치를 추정하는 방법
- **저온 기동 기술** (2024) — 극저온 환경에서 모터 기동 신뢰성을 확보하는 방법

또한 **EOP 12V 400W 전동기** 국책과제를 수행하며 SVPWM 절환 기법 연구, 온도 조건별(55°/85°/120°) 열평형 시험, 다이나모미터 개발까지 모터 제어의 기술적 깊이를 쌓았습니다.

### "제품 전체"를 보기 시작하다

전환점은 **PLUVA FRIEND SS500** 프로젝트였습니다.

SS500은 농업용 전동 자율주행 Speed Sprayer — 48V 460Ah LiFePO4 배터리로 구동되는 과수원 방제 로봇입니다. 겉으로 보면 농기계지만, 안을 들여다보면:

- **CAN 기반 분산 제어 시스템**: VCU(STM32) → 구동모터 × 2 → 팬모터 → 펌프 → BMS, 5개 CAN 노드가 실시간으로 통신
- **3가지 제어 모드**: 리모콘(RC), LCD 패널, 자율주행(ADT) — 각각의 제어권 전환 시 안전을 보장하는 상태머신
- **자율주행 스택**: RTK-GPS(1~2cm 정밀도), 카메라, IMU 기반 경로 추종
- **클라우드 연동**: AWS OTA 업데이트, 작업 데이터 통합 관리

이것은 농기계가 아니라, **Physical AI 제품**입니다.

저는 이 프로젝트에서 모터제어 개발자(본업)와 주니어 PM(겸임)을 동시에 수행했습니다.

### 기술 깊이 — 모터제어 개발자로서

| 영역 | 구체적 성과 |
|------|------------|
| **배터리 BMS 정합성** | SOC 162.4% 오버플로우, 셀 편차 260mV 이슈를 CAN 로그 분석으로 발견. 3차에 걸친 ROM 업데이트 검증 주도, 배터리 업체 WBS 수립 |
| **상태머신 설계** | RC/LCD/ADT 3모드 제어권 전환 로직 설계. NONE 상태 삭제로 안전 버그 해결. 모든 Transition에서 Init(속도 0) 루틴 적용 |
| **안전 시스템** | 범퍼 정지거리 0.082m(3km/h), 상전류 피크 308A, 정착시간 467ms 검증. Main Relay 소손 → "VCU 정지 → 5분 후 MR 차단" 안전 시퀀스 재설계 |
| **펌프 전력 분석** | 현재 0.96kW → 타겟 3.23kW 전력 예산 분석. 비용 3배 증가(30만→130만원), 운전시간 반감 트레이드오프 정량화 |
| **동파 RCA** | 5호기 동시 발생 → 솔레노이드 오링 가공불량(depth+wall thickness) 근본원인 추적 → 20개 발주 + 설계변경 |
| **CAN DB 설계** | DBC 파일 4종 직접 작성/관리. VCU↔모터컨트롤러↔BMS↔팬 다중 노드 통신 정합성 검증 |

### AI를 프로세스에 통합하다

저는 Claude Code를 단순 코딩 보조가 아니라, **개발 프로세스 자체에 통합**해서 사용합니다:

- **5개 에이전트 시스템** 설계/운영: PM_Lead, Engineer_1~3, Notion_Admin
- **이슈 관리 자동화**: Notion DB → 로컬 마크다운 동기화 → INDEX 자동 생성
- **주간보고 자동화**: /weekly-report 스킬로 MD + Excel + Notion 동시 출력
- **기술 분석**: CAN 로그 분석, 배터리 정합성 검토를 Claude와 페어로 수행

"AI를 사용하는 수준"이 아니라 "AI 워크플로우를 설계하는 수준"입니다.

## 왜 Physical AI인가

GINT는 "농기계 회사"가 아닙니다. Foresight(이스라엘 3D 비전) + KONEC(한국 Tier-1)과 자율주행 트랙터를 공동개발하고, 2026년부터 양산 판매를 시작하며, 2030년 $35M 매출을 전망하는 **Agricultural Robotics 회사**입니다.

SS500에서 경험한 것은:
- **모터 제어** → 물리적 액추에이터를 정밀하게 움직이는 기술
- **CAN 분산 제어** → 다수의 노드가 실시간으로 협조하는 시스템
- **상태머신 + 기능안전** → 사람 근처에서 동작하는 기계의 안전 설계
- **배터리 + 전력 관리** → 전동화 제품의 에너지 시스템
- **자율주행 통합** → AI가 물리 세계를 인지하고 행동하는 루프

이 모든 것은 로보틱스, 자율주행 차량, 스마트 디바이스 — **Physical AI가 확장되는 모든 영역**의 기반 기술입니다.

## 엔지니어링 철학

**일론 머스크의 개발 5단계 프로세스**를 실무에 적용합니다:

| 단계 | 원칙 | SS500 적용 사례 |
|------|------|----------------|
| 1. 요구사항을 의심하라 | 주어진 스펙을 그대로 받지 않는다 | 아세아텍 100A 벤치마킹 기준이 우리 타겟 고객에 맞는지 재검토 |
| 2. 삭제하라 | 불필요한 것을 제거한다 | 제어권 NONE 상태를 삭제하여 상태머신 복잡도 감소 + 안전 버그 해결 |
| 3. 단순화/최적화하라 | 남은 것을 더 단순하게 | CAN 메시지 표준화, 제어권 전환 시 Init 루틴 통일 |
| 4. 속도를 높여라 | 최적화된 것을 빠르게 | 배터리 정합성 검증 자동화 체크리스트 도입 |
| 5. 자동화하라 | 반복 작업을 자동화 | Claude Code로 이슈 관리, 주간보고, 기술 분석 자동화 |

## 지향점

**Physical AI 제품의 하드웨어 개발** — 로봇, 자율주행, 스마트 디바이스

하드웨어를 이해하면서 소프트웨어도 쓸 수 있는 브릿지 엔지니어로서,
AI가 물리 세계와 만나는 접점에서 **"실제로 동작하는 제품"**을 만들고 싶습니다.
