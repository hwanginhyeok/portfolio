# 역량 테마 매핑 — 프로젝트를 가로지르는 7가지 역량

> 목적: "프로젝트별"이 아닌 "이 사람이 반복적으로 잘하는 것"으로 포트폴리오를 재구성
> 기준: 2개 이상의 프로젝트에서 같은 결의 업무가 반복되면 하나의 테마로 묶는다

---

## 테마 전체 맵

```
프로젝트        T-1     T-2     T-3     T-4     T-5     T-6     T-7
              모터제어  CAN설계  PCB/HW  시험장비  데이터   양산품질  문제해결
─────────────────────────────────────────────────────────────────────
SCU (현대/DIC)   ■■■     ■■      ■■■              ■       ■■■

ADT 모터         ■■                       ■       ■■
SS500 (핵심)     ■■      ■■■              ■■■     ■■■     ■■■     ■■■
자동차 (WIA)     ■
특허             ■■■                              ■
```

**■ = 해당 테마에서 실질적 업무 수행**

## T-1. 모터제어 깊이 — "사다리꼴에서 FOC/SVPWM까지"

### 핵심 메시지
> 3년간 모터 제어 기술을 사다리꼴 → FOC → SVPWM으로 진화시키며, 특허 2건과 국책과제를 달성한 모터제어 전문가

### 프로젝트별 업무

| 프로젝트 | 시기 | 구체적 업무 | 성과 |
|----------|------|-----------|------|
| **SCU** | 2022~2023 | Infineon TC23x 기반 BLDC 사다리꼴 구동 구현, 게이트 드라이버(BTS3800SL) 설계 | PCB 3세대 진화(ES→AS→A2S) |
| **EOP 400W** | 2023~2024 | SVPWM 절환 기법 연구, FET 온도 16~17°C 저감 실증, 저온기동 특성 연구 | **국책과제 완료**, 특허 출원(저온기동) |
| **특허** | 2023 | 모터 초기 위치 검출 알고리즘 발명 (PN231067KR) | **특허 출원** |
| **ADT** | 2023 | 3세대 모터 개선활동, PI 파라미터 최적화, TVS Diode 적용 | Zbench 60%↑, OVT 제거 |
| **SS500** | 2025~2026 | CAN 기반 모터 컨트롤러 통합, 구동/팬/펌프 3종 모터 시스템 제어 설계 | 5노드 CAN 네트워크 정합성 확보 |

### STAR 1줄
사다리꼴 구동(SCU)으로 시작해 SVPWM(EOP)을 연구하고, 특허 2건(초기위치/저온기동)을 출원하며, CAN 기반 다중 모터 시스템(SS500)까지 — 모터 제어의 원리부터 시스템 통합까지 커버.

### 포트폴리오 위치
**기술 딥다이브 섹션** — 가장 깊은 기술 역량으로 첫 번째에 배치

---

## T-2. CAN 통신 설계 — "단일 노드에서 5노드 시스템까지"

### 핵심 메시지
> CAN DB 설계, 프로토콜 구현, 정합성 검증을 단일 노드에서 다중 노드 분산 시스템으로 확장한 경험

### 프로젝트별 업무

| 프로젝트 | 시기 | 구체적 업무 | 성과 |
|----------|------|-----------|------|
| **SCU** | 2022~2023 | CANoe/BUSMASTER 기반 CAN DB 설계, DIC 협력사 CAN 프로토콜 협의 | CAN DB v1.0 작성, DBC/INI 파일 관리 |
| **EOP 400W** | 2023 | CAN Sleep/Wakeup 기능 구현 (MC9S12ZVMC), 저전력 대기 모드 설계 | rev0~4까지 5차 개선, 사내 보고 |
| **SS500** | 2025~2026 | 5노드 CAN 네트워크 설계 (VCU↔구동↔팬↔펌프↔BMS), DBC 4종 작성, 배터리 CAN 정합성 3차 검증 | 37건 이슈 중 CAN 관련 핵심 이슈 다수 해결 |

### 진화 스토리
```
2022: 단일 노드 CAN 통신 (SCU ↔ DIC 상위 ECU)
  ↓
2023: CAN 저전력 모드 (Sleep/Wakeup) — 배터리 드레인 방지
  ↓
2025: 5노드 분산 제어 시스템 (VCU 중심) — 실시간 정합성 검증
```

### STAR 1줄
SCU에서 자동차 표준 CAN 프로토콜을 배우고, EOP에서 Sleep/Wakeup을 구현하고, SS500에서 5노드 분산 시스템의 DBC 설계부터 정합성 검증까지 — CAN 통신의 전체 생명주기를 경험.

### 포트폴리오 위치
**기술 딥다이브 섹션** — 모터제어 다음, "시스템을 묶는 기술"로 배치

## T-3. PCB/HW 설계 — "매 세대 더 나은 보드를 만든다"

### 핵심 메시지
> 회로 설계 → PCB 레이아웃 → 시제 → 검증 → 양산의 HW 개발 전 과정을 3개 프로젝트에서 반복

### 프로젝트별 업무

| 프로젝트 | 시기 | 구체적 업무 | 성과 |
|----------|------|-----------|------|
| **SCU** | 2022~2023 | ES → AS → A2S **3세대 PCB 진화**, 회로도/실크/BOM 관리, Infineon TC23x + BTS3800SL + TLV5614 DAC 설계, Thermistor 온도 측정 | 자동차 환경시험(ES95400)/EMC(ES96200) 규격 통과 |
| **EOP 400W** | 2023~2024 | **6층 PCB** 설계, BOM 리스트 관리, HW 블록다이어그램, SPICE 시뮬레이션(게이트 드라이버, 전원) | 국책과제 완료보고서 PCB 섹션 작성 |
| **SS500** | 2025~2026 | VCU REV01 → REV02 → **REV03** 진화, STM32F479 기반, PIN MAP 관리, 4층 PCB, GERBER/SMT/DXF/BOM 일체 관리 | 양산용 VCU 확정 |

### 진화 스토리
```
SCU:  자동차 규격 PCB (EMC/환경시험 통과, Infineon MCU)
  ↓
EOP:  6층 고밀도 PCB (전력 변환 + 제어 + 통신 통합)
  ↓
SS500: 양산용 VCU (STM32, 3세대 리비전, BOM 연동)
```

### STAR 1줄
"설계 → 시제 → 검증 → 개선"을 3개 프로젝트에서 9세대(SCU 3 + EOP 1 + VCU 3) 반복하며, 자동차 규격 통과부터 양산 BOM 관리까지.

### 포트폴리오 위치
**기술 역량 맵** — "SW만 아는 사람이 아님"을 증명하는 HW 역량

### 포트폴리오 위치
**엔지니어링 철학 섹션** — "이 사람의 사고방식"을 보여주는 핵심 스토리

## T-5. 데이터 기반 의사결정 — "감으로 하지 않고 측정한다"

### 핵심 메시지
> 모든 기술 의사결정을 정량적 데이터에 기반하여 수행. "대충 될 것 같다"를 허용하지 않는다.

### 프로젝트별 업무

| 프로젝트 | 의사결정 | 데이터 근거 |
|----------|---------|------------|
| **EOP** | SVPWM 채택 | FET 온도 16~17°C 저감 (2,932개 데이터 비교) |
| **ADT** | 3세대 모터 개선 승인 | Zbench 2.714→4.346 (+60%) |
| **SS500** 배터리 | 업체 ROM 재수정 요구 | SOC 162%, 셀 편차 260mV 정량화 |
| **SS500** 펌프 | 80A급 업그레이드 보류 | 비용 3배(130만원), 운전시간 4h→2h 정량 분석 |
| **SS500** 팬 | 20파이 프로펠러 채택 | 출력 19.7→42.2W (+114%) |
| **SS500** 범퍼 | 자율주행 안전 검증 | 정지거리 0.082m, 피크 308A |
| **SS500** 성능 | SS500 포지셔닝 | 압력 2배(30 vs 15bar), 제어성 우위 |

### STAR 1줄
펌프 업그레이드를 "비용 3배 + 운전시간 반감"으로 정량화해서 경영진 판단을 도운 것부터, SOC 162%를 셀별 전압까지 파서 업체에 전달한 것까지 — 데이터가 의사결정의 언어.

### 포트폴리오 위치
**T-4(시험장비)와 연결** — "만들고(T-4) → 측정하고(T-5) → 결정한다"의 흐름

## T-6. 양산 품질 프로세스 — "프로토타입과 양산은 다른 세계"

### 핵심 메시지
> 자동차 규격(IATF 16949)부터 농기계 종합검정까지, 프로토타입을 양산 품질로 끌어올리는 프로세스 경험

### 프로젝트별 업무

| 프로젝트 | 시기 | 구체적 업무 | 성과 |
|----------|------|-----------|------|
| **SCU** | 2022~2023 | 자동차 환경시험(ES95400), EMC(ES96200) 규격 적용 및 통과, DIC/현대 품질 기준 대응 | 자동차 Tier-1 납품 규격 경험 |
| **SS500** | 2025~2026 | APQP Phase 2~3 수행, DFMEA/DRBFM, BOM 132개 항목 관리, 양산 초도품 V0.0.0 → PreMP V0.3.4, 금형 발주/관리, 이슈 37건 트래킹, NCR 보고서 | 양산 16+4대 준비 |

### 관리 역량 수치

| 항목 | 수치 |
|------|------|
| APQP 단계 | Phase 1~3 직접 수행 (5단계 중) |
| 이슈 관리 | 37건+ (10개 카테고리, Notion DB) |
| 전력제어 담당 | 14건 (진행 7, 보류 2, 시작전 2, 완료 3) |
| BOM 관리 | 132개 부품 항목 |
| 재고 미확보 식별 | 23건 |
| 양산 계획 | 16pcs + 대차 2 + Spare 2 |

### STAR 1줄
SCU에서 자동차 품질 기준(ES95400/EMC)을 배우고, SS500에서 APQP/DFMEA/BOM/이슈관리를 운영하며 프로토→양산 전환을 주도.

### 포트폴리오 위치
**PM 경험 섹션** — 기술자가 PM도 할 수 있다는 크로스펑셔널 증명

## T-7. 문제 해결 (RCA) — "현상→원인→대책→검증"

### 핵심 메시지
> 필드에서 발생하는 문제를 감이 아닌 체계적 근본원인 분석(Root Cause Analysis)으로 해결

### SS500 RCA 사례 (3건)

| # | 현상 | 원인 추적 | 대책 | 검증 |
|---|------|----------|------|------|
| 1 | **동파** (5호기 동시 발생) | 솔레노이드 파일럿 제어부 오링 가공불량 (depth + wall thickness) | 20개 발주 + 배출부 볼밸브 추가 + 정온전선 설치 | 교체 후 재발 없음 |
| 2 | **Main Relay 소손** | 비상정지 시 MR 즉시 차단 → 모터 역기전력에 의한 스위칭 회로 손상 | "VCU 정지명령 → 5분 대기 → MR 차단" 안전 시퀀스 재설계 | TLE8110ED 기반 재현 시험 계획 |
| 3 | **SOC 162%** | 배터리 BMS ROM의 SOC 계산 로직 오류 + 보호 Flag 미구현 | CAN 정합성 검증 시트 자체 제작, 업체 WBS 수립, 3차 반복 검증 | Alpha 1,2,3호기 동시 검증 완료 |

### RCA 루프 패턴
```
현상 발생 → 가설 수립 → 데이터 수집(CAN 로그, 오실로스코프, 목시)
  → 원인 특정 → 대책 설계 → 실시 → 재발 모니터링
```

### STAR 1줄
동파 5호기 동시 발생에서 오링 가공불량까지 추적하고, Main Relay 소손에서 비상정지 시퀀스를 재설계하고, SOC 162%에서 3차 반복 검증 프로세스를 수립 — 매번 근본까지 파고든다.

### 포트폴리오 위치
**기술 딥다이브** 또는 **엔지니어링 철학** — 문제 해결 역량은 어느 섹션이든 강력

## 테마 기반 포트폴리오 재구성 제안

기존 TASK의 "프로젝트별" 구조를 "테마별"로 전환할 수 있습니다:

**현재 (프로젝트 중심)**:
```
2-1. SS500 프로젝트 개요
2-2. SS500 기술 딥다이브
2-3. SS500 PM 경험
2-5. 사이드 프로젝트
```

**제안 (테마 중심)**:
```
섹션 1. About Me + 커리어 내러티브 ← D-1 (완료)
섹션 2. 기술 깊이
  2-a. 모터제어 (T-1): SCU→EOP→SS500 진화 + 특허 2건
  2-b. CAN 시스템 (T-2): 단일→Sleep→5노드
  2-c. HW 설계 (T-3): 9세대 PCB 진화
섹션 3. 엔지니어링 마인드셋
  3-a. 시험장비 (T-4): D-12 (완료)
  3-b. 데이터 기반 의사결정 (T-5)
  3-c. 문제 해결 RCA (T-7): D-2 배터리 (완료) + 동파/MR
섹션 4. 제품을 만드는 사람
  4-a. 양산 품질 (T-6): APQP + 이슈 37건 + BOM 132개
  4-b. AI 워크플로우: 5-에이전트 시스템
섹션 5. Where I'm Going
  5-a. Physical AI 비전
  5-b. 학습 로드맵
```

**장점**: 같은 사실이지만, "SS500에서 이것저것 했다"가 아니라 **"이 사람은 이런 역량을 가지고 있다"**로 읽힌다.
