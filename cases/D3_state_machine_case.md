# Case Study: 제어권 상태머신 — 안전한 모드 전환의 설계

> 프로젝트: PLUVA FRIEND SS500 (GT-SS500)
> 역할: 전력제어개발팀 선임연구원 (상태머신 로직 검토/설계/검증)
> 기간: 2026.01 ~ 2026.02
> 이슈 번호: ISSUE #79

---

## Situation

SS500은 3가지 제어 모드를 지원합니다:

| 모드 | 입력 장치 | 사용 상황 |
|------|----------|----------|
| **RC** | RF 리모콘 | 근거리 수동 조종 (주 사용 모드) |
| **LCD** | 본체 디스플레이 패널 | 현장 세팅, 파라미터 조정 |
| **ADT** | 자율주행 PC (GNSS+카메라+IMU) | 자동 경로 추종 방제 |

문제는 **이 모드들 사이를 전환할 때 안전을 어떻게 보장하는가**입니다.

농기계는 비전문가(농민)가 운용합니다. RC로 조종하다가 LCD로 바꾸거나, 자율주행 중에 리모콘 페어링이 되거나, 비상정지 후 복귀할 때 — 모든 전환에서 모터가 예상대로 동작해야 합니다. 예상과 다르게 동작하면 사고입니다.

---

## Task

**핵심 질문**: "어떤 모드에서 어떤 모드로 전환하든, 차량이 예측 가능하게 동작하는가?"

구체적 검증 항목:
1. RC → LCD 전환 시 구동모터 속도가 초기화되는가?
2. LCD → ADT 전환 시 자율주행이 정상 시작되는가?
3. ADT → RC 강제 전환 시 기존 자율주행 속도가 잔존하지 않는가?
4. 비상정지 후 모든 모터 상태가 안전하게 초기화되는가?

---

## Action

### 1단계: DM 로직 및 하위 제어기 상태머신 알고리즘 검토 (~01/15)

VCU의 DM(Decision Maker) 로직과 각 모터 컨트롤러의 상태머신을 코드 레벨에서 리뷰했습니다.

**발견한 치명적 버그**:

> RC 모드에서 속도 3km/h로 주행 중, 리모콘 페어링이 발생하면 제어권이 RC→LCD로 전환됩니다.
> 이때 LCD 모드에서는 속도 지령을 보내지 않기 때문에, **모터 컨트롤러의 상태머신에 이전 속도(3km/h)가 그대로 남아 있었습니다.**

차량이 LCD 모드로 전환되었는데 **아무 조작 없이 계속 전진**하는 상황.

**근본 원인 분석**:
```
RC 모드 (속도 3km/h 주행 중)
    ↓ 리모콘 페어링 발생
LCD 모드 진입
    ↓ LCD에서 속도 지령을 보내지 않음
모터 컨트롤러: "새 지령이 없으니 마지막 지령(3km/h) 유지"
    ↓
차량이 무인 상태로 계속 전진 ← 위험!
```

더 깊은 문제: 기존 설계에 **CTRL = NONE** 상태가 존재했습니다. "제어권 없음" 상태인데, 이 상태에서는 어떤 모듈도 모터를 제어하지 않아 마지막 속도가 유지됩니다.

### 2단계: 아키텍처 변경 설계 (~01/23)

두 가지 핵심 변경을 제안했습니다:

**변경 1 — NONE 상태 삭제**
```
Before: RC ↔ NONE ↔ LCD ↔ NONE ↔ ADT
After:  RC ↔ LCD ↔ ADT (항상 명시적 제어 모드)
```

NONE 상태를 삭제하면 "아무도 제어하지 않는" 공백이 사라집니다.

**변경 2 — 모든 Transition에서 Init 루틴 적용**
```
제어권 전환 감지
    ↓
모든 모터 컨트롤러에 속도 0(Init) 지령 인가
    ↓
새 제어 모드 활성화
    ↓
새 모드에서 속도 지령 시작
```

LCD 제어권일 때는 **속도 0을 지속적으로 인가**하여, 과거 속도가 잔존하지 못하도록 했습니다.

### 3단계: 코드 수정 및 실차 검증 (01/26 ~ 02/20)

| 일정 | 작업 | 결과 |
|------|------|------|
| 01/26 | 민지선임 코드 배포 (Init 로직 반영) | 완료 |
| 01/29 | 코드 수정 완료 | NONE 삭제 + Init 적용 |
| 01/30 | 실차 검증 #1: RC ↔ LCD | 정상 — 전환 시 속도 0 확인 |
| 02/13 | 실차 검증 #2: ADT ↔ LCD, ADT ↔ RC 비정상 전환 | 정상 |
| 02/19 | 제어권 변경 Init 테스트 완료 | 모든 전환 시나리오 통과 |

**특이사항 발견 (02/20)**:
> RC 모드에서 속도 고정 설정 상태로 자율주행에 진입했다가 해제할 경우, 기존 속도 고정 설정이 유지됨

→ 이 동작은 의도된 것인지 확인 필요로 기록. 안전 위험은 없음 (속도 고정 해제 시 Init 동작).

## Result

### 정량적 성과

| 항목 | Before | After |
|------|--------|-------|
| 제어 상태 | 4가지 (RC, LCD, ADT, **NONE**) | 3가지 (RC, LCD, ADT) |
| Transition 안전성 | 이전 속도 잔존 가능 | **모든 전환에서 Init(속도 0) 보장** |
| 무인 전진 위험 | 존재 (NONE 상태) | **제거** |
| 검증 시나리오 | 미검증 | RC↔LCD, ADT↔LCD, ADT↔RC **6개 전환 조합 실차 검증** |
| 검증 기간 | - | 01/15 ~ 02/20 (5주) |

### 비즈니스 임팩트

1. **사용자 안전**: 모드 전환 시 차량이 예측 불가하게 움직이는 위험 제거
2. **자율주행 신뢰성**: ADT↔수동 전환이 안전하게 되어야 자율주행 기능의 실용성 확보
3. **양산 품질**: 16대 양산 전 안전 아키텍처 확정

## 면접 예상 Q&A

**Q: NONE 상태를 삭제하는 건 단순해 보이는데, 왜 중요한가요?**

> 단순한 변경이지만, 이것이 제1원칙 사고입니다. "제어권이 없는 상태가 왜 필요한가?"라는 본질적 질문을 했고, 답은 "필요 없다"였습니다. NONE 상태는 초기 설계에서 "아직 결정 안 됨"을 표현하기 위해 추가한 것인데, 런타임에서는 위험 요소가 됩니다. 복잡한 솔루션이 아니라, 불필요한 것을 삭제함으로써 안전성과 단순성을 동시에 얻었습니다.
