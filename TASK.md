# TASK — 취업용 포트폴리오

> **관리 룰:**
> - 상태: `예정` → `진행` → `완료`
> - 중요도: P1(긴급) > P2(중요) > P3(향후)
> - **완료 이동 (필수)**: 작업이 완료되면 **즉시** 작업 현황에서 삭제하고 완료 섹션 **최상단**에 추가한다.
> - 작업 현황 컬럼: `# | 분야 | 작업 | 중요도 | 상태 | 비고`
> - 완료 섹션 컬럼: `# | 분야 | 작업 | 완료일`
> - **넘버링 룰**: `{분야코드}-{순번}`
>   - `1-x`: 기획 — 포트폴리오 구조, 형식, 타겟 정의
>   - `2-x`: 프로젝트 케이스 — HIH_2(GT-SS500) 기반 프로젝트 정리
>   - `3-x`: 콘텐츠 — 자기소개, 기술스택, 철학, 커리어 내러티브
>   - `4-x`: 제작 — 실제 포트폴리오 제작 (웹/PDF/노션 등)
>   - `5-x`: 부가자료 — 이력서, 커버레터, GitHub 정비
>   - `6-x`: 사진/미디어 — 제품 사진, 촬영, 이미지 소재
> - 정렬 순서: **분야별 → 날짜별**
>
> 최종 수정: 2026-04-04 (v1.9) — 이미지 적용, 다이어그램, 구조 개편

## 금일 세션(2026-04-04) 변경 사항 요약

### 이미지
- HIH_2에서 49개 이미지 수집 → WebP 변환 (69MB → 13MB, 86% 압축)
- PPTX 3개에서 이미지 추출 (전장함 CAD, 노즐 분사 시험, EOP 알고리즘)
- 제품 부품 사진 5장 제거 (시험 환경 아닌 것)
- PCB 이미지 전부 제거 (본인 작업 아님)

### 페이지 구조
- 전 케이스 페이지 **문제 → 결과 → 해결과정** 두괄식으로 통일
- `eop-svpwm` → `eop-400w` 경로 변경 + 타이틀 변경
- EOP에 초기위치 검출 특허 섹션 추가 (4개 성과 구조)
- CAN Sleep 섹션 재작성: IC/MCU 레지스터 분석 + ISO 11898 강조
- 극저온 기동: EEMF estimator + Position/Speed estimator 블록도 추가

### 다이어그램
- 홈 시스템 아키텍처: Mermaid SVG → HTML/CSS `SystemArchitecture.astro` 컴포넌트
- Mermaid SVG 3종 렌더링 완료 (아직 상태머신 페이지에서 사용 중)

### 수치 이슈
- **FET 온도 16~17°C 저감**: 데이터 검증 불가 → 현재 "1~3°C"로 변경
- **효율 1%/4%**: 경력기술서 기준, 본인 확인 필요
- 원본 CSV 분석 결과: 1Assy vs 2Assy 차이 0~4°C (SVPWM vs DPWM 비교 아님)
